# <module> Spec（模板）

## 1) Purpose
- 负责 / 不负责：清晰界定范围。

## 2) Inputs
- 来源、格式、字段、约束（引用 contract_input）。

## 3) Outputs
- 去向、格式、字段、稳定性（引用 contract_output）。

## 4) Process
- 步骤列表或伪代码，含外部依赖调用。

## 5) Configuration
- 参数、默认值、范围、影响。

## 6) Performance Budget
- 延迟/吞吐/资源预算（可选，如无则声明 N/A）。

## 7) Error Handling
- 错误码/异常/降级策略。

## 8) Edge Cases
- 空输入/异常格式/极端数据。

## 9) Acceptance Criteria
- Given/When/Then + 样例输入输出；每条对应 tests。

## 10) Trace Links
- 指向 tests、contracts、traceability 行。
